New Process Wizard

Overview

The New Process Wizard lets you create a new business or callable sub process.

Figure 3.1. The New Process Wizard


Accessibility

File > New > Process

Features

Process Project

Choose the project where the new process should be created.

Process Group

Select a group where the new process will be inserted (this is roughly equivalent to a namespace). Select the <default> process group to create a process directly below the project's processes folder (i.e. equal to "no group"). You can click on the group folder button to open the New Process Group Wizard, if you want to create a new group "on the fly". The process groups are listed relative to the project's process folder.

Process Name

Enter the name of the new process.

Process Type

Standard Process : This option is the default option an creates a normal standard business process. Use this option to implement your business logic.

Callable Sub Process : This option creates a callable sub process including a process-call-start element and a process-call-end element. You need to implement your process between those two elements. It is allowed to have multiple Process Starts and Process End elements in a callable process.

Web Service Process : This option creates a web service process which can be called from other systems. WS Start and WS End elements will be created automatically and you can implement your process between these elements. Please note that no user interaction may occur in a web service process.

Process Data Policy

Use default: Select this option to use the project's default data class as data structure for the new process.

Use existing: Select this option to choose another, already existing data class as data structure for the new process.

Create new: Select this option to create a new, empty data class that will be associated with the new process.

Process Data Class

This text field shows the data class that will be associated with the new process (according to the selection in the Process Data Policy section).

If you've selected use existing as process data policy then you may click on the button to select an existing data class. It is strongly recommended to select a data class from the project where the process will be created in order to avoid dependencies on the implementation of another project.

If you've selected create new then you may enter the name of the new data class to create (including namespace). Initially a data class name that is based on the new process' name and group will be suggested, but you're free to change it.
