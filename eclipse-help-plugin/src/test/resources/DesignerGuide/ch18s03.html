Memory Problems

OutOfMemoryException: Java heap space

If this error occurs, then the Designer requests more memory than it is allowed to use. This can happen when a lot of data is used during the process simulations. Ensure, that your computer has enough memory. Then you can increase the maximal memory consumption of the Xpert.ivy Designer. Just open the XpertIvyDesigner.ini and change the value in the line -Xmx768m to a higher one.

OutOfMemoryException: PermGen space

The so-called permanent generation area in the memory of a Java application is filled up. This may happen when a load of interned strings are used and a lot of classes are loaded (e.g. by reflection). You may increase the maximal size of the permanent generation by editing the XpertIvyDesigner.ini and setting -XX:MaxPermSize=128m to a higher value.

Note

If you have a 64 Bit computer use the 64 Bit version of the Xpert.ivy Designer. This allows you to allocate more than 2 GB memory.
