Content Management System (CMS)

A content management system CMS is a hierarchically organized database for any content such as labels, small texts, images, source snippets, documents, etc. Content can be provided in different languages and may optionally define a validity period. Content is managed in a tree of so called content objects, which may contain multiple content object value children for different languages. A content object has a type (and all of it's values have that type, too) and a path which is expressed as an URI of the form /Labels/Common/CustomerName and which depicts the content object's location in the hierarchically organized content tree.

Tip

In Xpert.ivy, each project has its own CMS. Content is looked up by means of the above mentioned content object URI. If the look-up of an URI fails for the current project, then the CMS's of the required projects are searched with the same URI recursively. Using this mechanism, you can effectively inherit and share content from e.g. a base project that holds all the common texts and images.

By using a CMS consistently throughout a project, i.e. for all content which is displayed to the user, it is very easy to internationalize your applications. Simply add for each content object a value in the new language that you want to support.

Content is added, edited and organized with the CMS Editor.

Access CMS Content with a Browser

Normally the CMS Content is used in a HTML Dialog, a Rich Dialog or a Process. But some content objects like a page or a document can be accessed directly from the browser.

The URL to access CMS content is built as follows: http://<servername>:<port>/ivy/cm/<application name>/<process model>/<path in CMS>

Assumed you have created a page in CMS under /StaticContent/MyPage in a project named Test and the engine of the Xpert.ivy Designer is started, the following URL would display this page in the browser: http://localhost:8081/ivy/cm/designer/Test/StaticContent/MyPage.

Note

It is possible to access any page with this mechanism. But as most pages are used to display information fetched in a process it might lead to an error trying to display them without data.

Common Content Object Types

There are various types of content. For each content type the CMS contributes its specific editors to intuitively manipulate the content.

Table 8.1. Content Object Types

FolderFor organisational purposes only, they serve as container for other content objects. Page Editor
StringShort texts (single line), e.g. labels, names, descriptions, tool tips. String Editor
TextLonger texts (multi paragraph, multi line). Text Editor
ImageAn image of arbitrary size. GIF Image, PNG Image, JPG Image types are supported. Image Editor
SourceScripts of any form, e.g. javascript or jsp snippets. Source Editor
PageContainer object for HTML Page content. Normally created by the New HTML Page Wizard. HTML Page Editor
CSSCascaded Style Sheet definitions CSS Editor

PanelPanels are content parts for Page objects and are defined with Layout objects. HTML Panel Editor
