Content Editor

The Content Editor is used to manipulate content objects in the Content Management System (CMS) of a project.

The content editor consists of different parts that are described separately below:

  • CMS Tree

  • Content Object Details Area

  • Search / Index Bar

  • Selection of CMS

Generally, content objects are selected from the tree on the left hand side of the editor and edited in the details area on the right hand side. Multiple actions are available for modification of objects, both from the context menu and the tool bar.

Tip

Both the tree and the details area of the CMS editor can also be opened separately as views, see CMS Tree View and Content Object View for further information.

Accessibility

1. Xpert.ivy Project Tree -> double click on the entry in the tree.

2. Various editors for content objects -> click on the Content Smart Button

CMS Tree

The CMS tree is used to create, select, delete and rename content objects and their values.

The following actions are available from the context menu of the currently selected tree node:

Add content object...

Adds a new content object of the selected type as child of the currently selected content object. Select the type of the content object to add from the sub menu that opens up. The new content object is created with an empty value in the default language of the CMS.

Tip

The default language of a project's CMS can be set in the project properties under Xpert.ivy > Content Management

Add content object value...

Adds a new content object value of the selected language to the currently selected content object. Select the language from the sub menu that opens up. If the desired language does not exist yet, you can add it by clicking on the Other Language... menu entry. The new language will be added to the CMS and a new content object value will be created for it.

Rename

Opens a dialog where you can enter a new name for the currently selected content object.

Warning

When you rename a content object, the URI of all it's children will change (e.g. from /Labels/Common/Ok to /Labels/Buttons/Ok). Any references to those objects (including the renamed object) will not be updated automatically and might be broken!

Copy

Copies the currently selected content object (including all of it's children) to the clipboard. The copied objects can be inserted somewhere else in the content tree with Paste.

Paste

Inserts any content object(s) that were copied to the clipboard with Copy as children of the currently selected content object.

Note

Not all content object types are allowed as children of other content object types. As a result of such a restriction, the Paste menu entry might be disabled if the content in the clipboard is not suitable for insertion below the currently selected content object.

Delete

Deletes the currently selected content object (including all of its children) from the CMS after requesting a confirmation from the user.

Warning

Deleting a content object will break all references to the object or its children!

Copy URI

Copies the URI of the currently selected content object to the system clipboard. Use ctrl-v to insert the URI into any text fields or editors.

Copy URI as IvyScript macro

Copies the URI of the currently selected content object as an IvyScript macro tag to the system clipboard. Use ctrl-v to insert the macro into any editor that can expand IvyScript macro tags (blue background).

Show/Edit Details

Opens up a dialog which allows to enter/edit a description of the currently selected content object.

Refresh Content

Refreshes (i.e. reloads) the content below the currently selected content object.

Content Object Details

The Content Object Details area is used to view and edit values of a content object.

Information about the currently selected content object is displayed in the Content Object Meta Information section at the top of the editor: date and time of last change, author, total number of values, content type and path.

Each value of the selected content object is shown with an appropriate editor inside a collapsible section and is labelled with the language of the value. The default value is marked with a * (star) after the language indication. Information such as date and time of last change and author is also shown for each value.

Toolbar Actions

The following actions are available on the toolbars of the content object section and the various value editor sections:

Open page preview

Opens the page as preview in a web browser. If the Web Browser View is active it is used otherwise an external browser is opened to show the preview.

Add new content object value

Adds a new value to this content object. A dialog is opened for the user to select the language of the new value.

Rename content object

Opens a dialog where a new name for the this content object can be entered.

Warning

When you rename a content object, the URI of all it's children will change (e.g. from /Labels/Common/Ok to /Labels/Buttons/Ok). Any references to those objects (including the renamed object) will not be updated automatically and might be broken!

Copy content object / value

Copies this content object (including all of it's children) or content object value to the clipboard. The copied objects can be inserted somewhere else in the content tree with Paste.

Delete content object / value

Deletes this content object or content object value from the CMS. A content object is deleted with all of its values and child content objects.

Warning

Deleting a content object will break all existing references to the object or any of it's children!

Show/Edit Details of content object / value

Opens up a dialog which allows to enter/edit a description for this content object or content object value. Content object values may define a validity range and can be selected to be the default value (then the previous default value will be more be the default value).

Import value content

Opens a file dialog that allows to select a file with content to be imported.

Note

Not all content types allow to import content (e.g. strings do not). If import is not supported, then the toolbar action will be disabled.

The file selection dialog will only show files that are suitable for import, depending on the standard extension for the required content type. I.e. you can not select a .css file for import into a .png content object.

Save and Reload of Content Object Data

The data of the currently edited content object (i.e. the content of it's values) is saved automatically when a new content object is displayed, e.g. when selecting another content object in the CMS Tree of the content editor. The object is also saved when you click on Ok in the content editor. Last but not least, you may also press the Apply button, which will immediately persist the currently edited object and it's values while keeping the editor open.

If you wish to discard the changes that you made to the currently displayed object (including to any of it's values), then you can either click the Cancel button of the content editor (which will discard any changes and close the editor window) or press the Reset button. The latter will keep the editor window open and reload the current object's data from the database and thus reset it's values to the content that they had before you started editing.

Tip

You may also use this function if you wish to update the contents of a single content object, in case you know that it's values have changed in the database / file system since the time you started editing.

CMS Search

By using the CMS search you can filter the contents in the Content Management System tree according to your search string.

Search Field

Depending on the scope of your search (content, description or path) your search string may have a different format.

If you search in content , then you have to enter the keywords that you expect to find in the content of a CMS object. You can use a * (star) or a ? (question mark) as wild-cards to search for any sequence of characters or any single character, respectively. For example enter ivy* or image?data* to search for all content (i.e. words) that start with ivy or any content that contains a word starting with image then any character (e.g. a one-digit number) and finally with the term data.

Tip

You should get used to end all your queries with a * to match as many objects as possible containing your search string.

Warning

You can not use a * or a ? at the beginning of your search string if you search in content.

If you search in path , then you have to enter a expression that is part of the path of your content objects you are looking for. You can use a * (star) or a ? (question mark) as wild-cards to search for any sequence of characters or any single character, respectively.

Tip

When searching for paths it is allowed to use a * or ? in front of your query.

If you search in description , then you have to enter an expression that is part of the description of the content object you are looking for. You can use a * (star) or a ? (question mark) as wild-cards to search for any sequence of characters or any single character, respectively.

With a successful search, the CMS tree will be reduced to only show the content object values that match your search query.

Search Scope

The search scope combo box can be used to define the scope of the search (i.e. search in content object names or in (textual) content or in description.

Create / update index

After adding, renaming and deleting of content objects the search index might not be up to date anymore and wrong or missing search results might be the result. Click this button to update your local search index.

CMS Selection

The CMS selection combo box can be used to quickly switch to a CMS of a required project (with respect to the currently active project). This helps you to analyze what the scope of a CMS lookup, if it is sent from the current project. Content objects will always be searched in the current project first. If the lookup is not successful, then the required projects will be queried recursively until a match is found (see the Project Deployment Descriptor chapter for more information on this topic).

Note

You can also copy and paste contents of the CMS of a required project. However, be aware that if you select, for example, /Texts/Info/Welcome from a base project, then this content object may be shadowed by a content object with the same URI /Texts/Info/Welcome which is located closer to the requester in the CMS hierarchy.

Tip

The contents of the CMS Tree View will change when you select an editor that shows an entity of another project. But only, if this project is not in the dependency tree of the currently selected project. So, you are able to put all shared content for multiple projects in one CMS of a shared resources project and keep the CMS Tree View focused on that project.
