Import Xpert.ivy Modeller Processes

Overview

Processes exported in Xpert.ivy Modeller as BPMN XML Export can be imported into the Designer through the Xpert.ivy Modeller Process importer.

Figure 3.3. The Xpert.ivy Modeller Process importer


Accessibility

File > Import > Xpert.ivy Modeller Processes (*.xml)

Features

From directory

A directory containing the Modeller BPMN Files. It's allowed proved a directory structure.

File selection

Selection of any valid Modeller BPMN Files within the From directory

Into folder

A directory pointing to a process group. All imported processes will be stored in this process group. In case the specified folder does not exist, it will be created automatically.

Option: Create roles for lanes

If the Modeller BPMN Processes selected for import contain lanes, it's possible to create new roles from the lane names in Xpert.ivy Designer automatically trough the import by this option.

Mapping of Elements by the importer

Xpert.ivy Modeller and Xpert.ivy Designer are tools to serve different needs, hence it's no possible to map any process element of the Modeller to exactly one corresponding element in Designer through the importer. As a consequence, the importer follows to main goals:

  • Achieve as much recognition of the Modeller process as possible

  • Provide a good basis for further implementation of process design

The mapping follows following rules:

Pools and lanes

Position, size and labels of swimlanes are adopted from the Modeller process.

Position and size of process elements

Position and size of process element nodes as well as any waypoints of process arc are adopted from the Modeller process.

Process events

In general, all process events are mapped directly to an event in the Designer.

Sub processes

Sub processes are mapped to a Composite element in the Designer.

Gateways

Exclusive gateways are mapped to an Alternative. Any other getways are mapped to a Task Switch.

Tasks

In general, task process elements are mapped to a BPMN Task in the Designer. If possible, the process element is mapped to a specific element e.g. BPMN User Task or BPMN Manual Task.

The BPMN Task elements behave basically similar to a Composite element, so it is possible, to implement the behavior of this process element at a lower level without changing the high level appearance of the process.

Links to other processes

Links to other processes are mapped to a Subprocess Call in the Designer, the call target remains empty after import.

Text annotations

Text annotations are mapped as Annotation in the Designer.

Data objects

Data objects are mapped as Annotation in the Designer, to distinguish them from text annotations, the differ in size and color.

Sequence Flows

Sequence Flows are mapped as Connector in the Designer.

Message Flows

Message Flows are mapped as Message Flow in the Designer.
