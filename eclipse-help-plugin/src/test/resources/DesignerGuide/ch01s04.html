Xpert.ivy Perspectives

A perspective defines a set and layout of views and editors in the Workbench window. Xpert.ivy defines 3 specific perspectives.

  • Process Perspective to model business process diagrams

  • Engine Perspective to simulate and debug processes

  • Html Dialog Perspective to design Html Dialogs

  • Rich Dialog Perspective to design Rich Dialogs visually

The current perspective is displayed on the title bar of the window and it is highlighted at the upper right corner in the shortcut bar. You can open and switch to a specific perspective using the Window menu or the buttons in the shortcut bar at the upper right corner. You can change the perspectives as you like. Move views and editors around, close and display views and editors. In the Window menu you will find the function to Reset a Perspective to its default definition.

Process Perspective

This perspective is intended to be used for process development. It contains the Process Editor and several views that help you to design and develop processes. Xpert.ivy switches automatically to the Process Perspective when you open a process.

  • The Ivy Project View shows an outline of an Xpert.ivy project and allows to select and load other files (especially processes) into an editor.

  • The Process Outline View gives an overview about the structure and elements of a process.

  • The Problems View lists all problems that are currently known to the system, e.g. element configuration errors.

Figure 1.3. The Process Perspective


Process Engine / Simulation Perspective

This perspective is intended to be used when simulating and executing process applications. It contains the Process Editor and a Web Browser as well as several views that help you to analyze and debug a running application. Xpert.ivy switches automatically to the Engine Perspective when you hit the Start Engine/Simulate Button on the tool bar.

  • The Ivy Project View shows an outline of an Xpert.ivy project and allows to select and load other files (especially processes) into an editor.

  • The Runtime Log View outputs log messages produced during the execution of a process model.

  • The Process Performance View list statistics about the currently running process application.

  • The History View allows to inspect the state of elements at a specific point of time of their execution.

  • The Debug View shows the currently executing request threads.

  • The Problems View lists all problems that are currently known to the system, e.g. element configuration errors.

  • The Breakpoints View gives an overview of the registered breakpoints in the application.

  • The Variables View allows to inspect the current process data when execution is halted at a breakpoint.

  • The Expressions View permits to execute Ivy Script expressions inside the current execution context.

Figure 1.4. The Engine Perspective


Html Dialog Perspective

This perspective is intended to be used for the development of Html Dialogs. It contains the Html Dialog Editor and several views that help you to design and develop processes. Xpert.ivy switches automatically to the Html Dialog Perspective when you open the view of a Html Dialog.

  • The Ivy Project View shows an outline of an Xpert.ivy project and allows to select and load other files (especially processes) into an editor.

  • The Outline View gives an overview about the structure and elements of the Dialog.

  • The Html Dialog Editor allows to edit Html Dialog views (.xhtml files)

  • The Properties View shows all (Java) properties of the currently selected panel or widget.

  • The Problems View lists all problems that are currently known to the system, e.g. element configuration errors.

Figure 1.5. The Html Dialog Perspective


Rich Dialog Perspective

This perspective is intended to be used for Rich Dialog development. It contains the Rich Dialog Panel Editor and several views that help you to design and develop the UI of Rich Dialogs. Xpert.ivy switches automatically to the Rich Dialog perspective when you open a Rich Dialog.

  • The Rich Dialog Panel Editor allows to edit Rich Dialog panels visually in a WYSIWYG fashion.

  • The Ivy Project View shows an outline of an Xpert.ivy project and allows to select and load other files (especially Rich Dialogs) into an editor.

  • The Java Beans View gives an overview about the structure and widgets of a Rich Dialog.

  • The Ivy Outline View displays mappings and bindings between processes/data and Rich Dialog widgets.

  • The CMS Tree View shows all elements of the currently selected Content Management System.

  • The Properties View shows all (Java) properties of the currently selected panel or widget.

  • The Problems View lists all problems that are currently known to the system, e.g. element configuration errors.

  • The Event Mappings View shows all registered event mappings for a Rich Dialog panel.

  • The Data Bindings View shows all registered data bindings between panel widgets and process data.

  • The Widget Configuration View facilitates editing and configuration of widget parameters.

Figure 1.6. Rich Dialog Perspective


Other

There are further perspectives from the eclipse platform that you may use for specific development tasks.

Resources Perspective

With the resource perspective you can explore and manipulate resources of the project at file system level.

Warning

Work carefully, since refactoring of Xpert.ivy resources is not supported under this perspective.

Team Synchronization Perspective

In the team synchronisation perspective you can browse through the changes between your local working copy and the base revision on the Subversion server.

Java Perspective

The Java perspective combines views that you would commonly use while editing Java source files.
